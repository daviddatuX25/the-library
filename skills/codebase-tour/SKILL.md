# Codebase Tour

## Steps

1. Query Honcho: "what areas of the codebase has the user not
   had explained to them yet?"

2. If Honcho has no record or everything is covered:
   ask me which file, feature, or area I want to understand.

3. Read the relevant files.

4. Explain as if onboarding a junior developer:
   - What problem does this code solve?
   - What pattern is being used and why this one?
   - How does it connect to other parts of the system?
   - What would break if this was written differently?
   - What's the technical debt here (if any)?

5. After explaining, check understanding:
   "What questions do you have? Anything feel unclear?"

6. Give a micro-exercise:
   "Without looking at the code, explain back to me what
   [key concept] does in this area."

7. Log to Honcho:
   - What area was explained
   - What patterns/concepts were covered
   - What I understood vs struggled with
   - Suggested follow-up topics
